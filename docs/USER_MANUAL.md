## 1. 快速入门

### 1.1 什么是 ProxySQL Admin WebUI

ProxySQL Admin WebUI 是一个现代化的 **Web 图形化管理界面**，用于管理 [ProxySQL](https://proxysql.com/) 数据库代理服务器。它通过 ProxySQL 的 MySQL 管理端口（默认 6032）提供配置可视化、实时监控和运维操作。

**核心能力：**

- 无需手写 SQL 即可完成配置（通过 63 个向导）
- 一键部署完整 ProxySQL + MySQL 代理架构（快速部署模板）
- 实时监控连接数、QPS、连接池等关键指标
- 管理多个 ProxySQL 实例
- 直接浏览和管理后端 MySQL 数据库
- 支持 MySQL 和 PostgreSQL 后端
- 完整的 DISK/MEMORY/RUNTIME 三层配置同步

### 1.2 系统要求

| 项目 | 最低要求 | 推荐配置 |
|------|---------|---------|
| 操作系统 | Linux (任何发行版) | Ubuntu 22.04+ / Debian 12+ |
| Python | 3.10+ | 3.12+ |
| Node.js | 24+ (仅开发模式) | 24+ (latest LTS) |
| 内存 | 512 MB | 2 GB+ |
| 磁盘 | 100 MB | 1 GB+ |
| ProxySQL | 2.7+ | 3.0+ |

### 1.3 安装方式

#### 方式一：Docker 部署（推荐生产环境）

```bash
docker pull ghcr.io/jacksonjiangxh/proxysql-admin-webui:latest
cp .env.example .env
# 编辑 .env 设置密钥
docker compose up -d
# 访问 http://localhost:8080
```

#### 方式二：裸机运行

```bash
make install
make build-frontend
make run
# 访问 http://localhost:8080
```

#### 方式三：开发模式（热重载）

```bash
make install
# 终端 1：make dev-backend  → :8080
# 终端 2：make dev-frontend → :5173
```

### 1.4 首次登录与修改密码

1. 打开浏览器访问 `http://localhost:8080`
2. 使用默认凭证登录：
   - 用户名：`admin`
   - 密码：`admin123`
3. 登录后导航到「用户管理」页面修改密码

## 2. 仪表盘

仪表盘是登录后的默认首页，提供 ProxySQL 运行状态的实时概览。

### 2.1 核心指标

- **活跃连接数**：当前正在使用的后端连接数
- **空闲连接数**：连接池中可用的空闲连接
- **总查询数**：ProxySQL 已处理的查询总数
- **QPS**：每秒查询数（Queries Per Second）

### 2.2 连接池状态

显示每个主机组中各后端服务器的连接池详情：
- 主机组编号
- 主机地址和端口
- 已用/空闲连接数

### 2.3 数据更新

仪表盘通过 **WebSocket** 实时推送数据，无需手动刷新。数据每 10 秒自动更新。

### 2.4 常见问题

**仪表盘显示空白？**
- 确保已在「服务器管理」页面配置了 ProxySQL 服务器
- 确保选中的 ProxySQL 服务器可正常连接

## 3. 服务器管理

### 3.1 功能概述

服务器管理用于添加、编辑、删除和测试 ProxySQL 管理接口的连接配置。支持管理多台 ProxySQL 实例。

### 3.2 添加服务器

1. 点击「添加服务器」
2. 填写以下信息：
   - **名称**：便于识别的服务器名称
   - **主机**：ProxySQL 管理接口的 IP 地址
   - **端口**：管理端口（默认 6032）
   - **管理员用户**：ProxySQL 管理用户名
   - **管理员密码**：ProxySQL 管理用户密码
3. 点击「测试」验证连接
4. 点击「保存」

> **重要**：ProxySQL 内置的 `admin` 用户默认仅接受本地连接（localhost）。如果 WebUI 和 ProxySQL 不在同一台机器，需要在 ProxySQL 中创建额外的远程管理用户。

### 3.3 切换当前服务器

在顶部栏的服务器选择器中点击要管理的实例，所有功能（仪表盘、向导、表浏览器等）将自动切换到该实例。

## 4. 配置向导

### 4.1 功能概述

配置向导是核心功能，提供 **63 个引导式表单**（W01-W63），无需手写 SQL 即可完成所有 ProxySQL 配置操作。

### 4.2 向导分类

| 分类 | 编号范围 | 说明 |
|------|---------|------|
| 后端服务器管理 | W01-W08 | 添加/编辑/批量导入后端 MySQL/PgSQL 服务器 |
| 用户认证管理 | W09-W15 | 创建/管理 ProxySQL 用户、前后端角色分离 |
| 查询路由规则 | W16-W23 | 读写分离、缓存、重写、镜像、限流、日志 |
| 复制与集群拓扑 | W24-W28 | 主从、MGR、Galera、Aurora、PgSQL 复制 |
| 系统配置 | W29-W42 | 连接池、查询处理、Admin 用户、集群节点 |
| 防火墙与安全 | W43-W45 | 用户/规则白名单、SQL 注入防护 |
| 运维与配置同步 | W46-W52 | Apply、Save、备份恢复、磁盘加载 |
| 监控与诊断 | W53-W63 | 查询分析、命令统计、连接池监控、集群状态 |

### 4.3 使用向导

1. 在配置向导页面选择一个向导
2. 填写表单参数（每个字段都有帮助文本说明含义）
3. 点击「预览 SQL」查看将执行的 SQL 语句
4. 确认无误后点击「执行」
5. 可选：勾选「自动应用」和「自动保存到磁盘」

### 4.4 常见工作流

详细的向导组合工作流请参见 [WIZARD_GUIDE.md](WIZARD_GUIDE.md) 第 5 节。

## 5. 快速部署模板

### 5.1 功能概述

快速部署模板提供**一键配置**完整的 ProxySQL + MySQL 代理架构。选择架构模式后，模板会引导你逐步配置所有必要组件，高级参数自动填入推荐值。

### 5.2 支持的架构模式

| 模式 | 说明 |
|------|------|
| 单主从复制 | 传统一主一从/多从异步复制架构 |
| 多主多从复制 | 多写入节点+读取副本架构 |
| MGR 单主模式 | MySQL Group Replication 单主模式，自动故障转移 |
| MGR 多主模式 | MySQL Group Replication 多主模式，所有节点可写入 |
| Galera 集群 | Galera/PXC 多主同步复制集群 |

### 5.3 使用流程

1. 导航到「快速部署」页面
2. 选择你的 MySQL 架构模式
3. 填写**全局配置**（监控用户、连接池参数等），这些配置会自动继承到后续步骤
4. 按步骤填写各组件配置（后端服务器、用户、读写分离、监控、连接池）
5. 可选跳过不需要的步骤
6. 点击「一键提交全部」生成并执行所有 SQL

### 5.4 全局配置继承

模板中的监控用户名/密码、主机组编号等共享参数只需填写一次，会自动继承到所有相关步骤。继承的字段会标记「已从全局配置自动继承」并可修改。

## 6. 表浏览器

### 6.1 功能概述

表浏览器用于查看和编辑 ProxySQL 的所有配置表。表按层级分类显示。

### 6.2 层级分类

| 层级 | 说明 | 包含的表 |
|------|------|---------|
| Disk | 磁盘层持久化配置 | `disk.mysql_servers` 等 |
| Memory | 内存层工作副本 | `mysql_servers`、`mysql_users` 等 |
| Runtime | 运行时实际生效配置 | `runtime_mysql_servers` 等 |
| Stats | 统计信息表 | `stats_mysql_query_digest` 等 |
| Monitor | 监控相关表 | `monitor.mysql_server_connect_log` 等 |
| Stats History | 统计历史表 | `stats_history.*` 等 |
| Other | 其他表 | 调度器、REST API 路由等 |

### 6.3 操作功能

- **搜索**：按关键字过滤行数据
- **排序**：点击列头排序
- **分页**：大量数据分页浏览
- **内联编辑**：直接点击单元格修改值
- **导出**：支持 CSV 和 JSON 格式导出数据

### 6.4 小白说明

每个表都有通俗易懂的中文说明，点击表名后自动显示，解释该表的用途，无需数据库背景。

## 7. SQL 查询控制台

### 7.1 功能概述

SQL 控制台提供专家模式，支持直接对 ProxySQL 管理接口执行 SQL 语句。

### 7.2 查询目标

- **Admin**：ProxySQL 管理命令（默认）
- **MySQL**：通过 ProxySQL 转发到后端 MySQL
- **PostgreSQL**：通过 ProxySQL 转发到后端 PostgreSQL

### 7.3 查询历史

- 所有执行的 SQL 自动保存到历史记录
- 支持按日期范围过滤
- 支持搜索历史 SQL 文本
- 支持重新执行历史查询
- 支持删除单条或清除全部历史
- 分页加载大量历史记录

### 7.4 数据导出

查询结果支持导出为 CSV 或 JSON 格式。

### 7.5 安全注意事项

- SQL 控制台会执行发送的所有语句，包括 DROP、DELETE 等危险操作
- 建议先在表浏览器中查看当前配置，再在控制台中修改
- 查询历史存储在本地数据库中

## 8. 数据库管理

### 8.1 功能概述

数据库管理功能允许你**直接浏览和管理** ProxySQL 管控的后端 MySQL 数据库。无需额外安装 MySQL 客户端，所有操作在 WebUI 中完成。

### 8.2 凭据来源

连接后端 MySQL 使用的凭据从 ProxySQL 配置自动获取：
- **主机和端口**：来自 `mysql_servers` 表
- **用户名和密码**：来自 `mysql_users` 表（仅 `backend=1, active=1` 的业务用户）

> **注意**：使用业务用户凭据而非监控用户（`mysql-monitor_*`），确保有足够的权限进行数据库操作。

### 8.3 操作流程

1. 导航到「数据库管理」页面
2. 从左侧选择后端 MySQL 服务器
3. 选择要使用的业务用户（用户名从 `mysql_users` 表获取）
4. 选择数据库 → 选择表
5. 进行以下操作：
   - **浏览**：分页查看表数据
   - **Schema**：查看表结构（字段、类型、键、默认值等）
   - **SQL 控制台**：执行任意 SQL 语句

### 8.4 SQL 控制台

- 支持 SELECT、DDL、DML 等所有 SQL 操作
- 显示执行结果（行数、耗时）
- 自动分页显示大量查询结果
- 示例 SQL 帮助快速上手

### 8.5 用户切换

可以随时切换不同的业务用户连接后端。切换用户后会自动重新加载数据库列表。

## 9. 配置同步

### 9.1 三层配置模型

ProxySQL 使用三层配置架构：

```
DISK（磁盘层） ← LOAD ← MEMORY（内存层） → APPLY → RUNTIME（运行时层）
                → SAVE →
                ← DISCARD ←
```

| 层级 | 说明 |
|------|------|
| **DISK** | 持久化存储（SQLite 数据库），重启后保留 |
| **MEMORY** | 内存中的工作副本，你编辑的地方 |
| **RUNTIME** | 实际生效的配置，ProxySQL 正在使用的 |

### 9.2 同步操作

| 操作 | 按钮 | 说明 |
|------|------|------|
| **Apply** | 全部应用到运行时 | MEMORY → RUNTIME，修改立即生效 |
| **Save** | 全部保存到磁盘 | MEMORY → DISK，重启后保留 |
| **Discard** | 丢弃更改 | RUNTIME → MEMORY，放弃未应用的修改 |
| **Load** | 从磁盘加载 | DISK → MEMORY，恢复磁盘中的配置 |

### 9.3 同步状态

配置同步页面显示每个模块的 Memory 和 Runtime 行数对比：
- 绿色「已同步」：Memory 和 Runtime 一致
- 黄色「未应用」：Memory 中有未推送到 Runtime 的更改

### 9.4 推荐操作顺序

```
修改配置 → Apply（生效）→ 验证 → Save（持久化）
```

## 10. 配置差异

### 10.1 功能概述

配置差异对比功能可视化展示 Memory 层和 Runtime 层之间的配置差异，帮助你识别未应用的更改。

### 10.2 差异信息

- 每个表的 Memory 行数和 Runtime 行数
- 「仅在 Memory」和「仅在 Runtime」的行数
- 已同步/未同步状态标识

### 10.3 使用场景

- 配置修改后检查哪些更改尚未生效
- 排查配置不一致的问题
- 在 Apply 之前确认影响范围

## 11. 配置备份

### 11.1 功能概述

配置备份功能用于创建、管理和恢复 ProxySQL 配置的完整快照。

### 11.2 操作功能

| 操作 | 说明 |
|------|------|
| **创建备份** | 创建当前 ProxySQL 配置的完整快照 |
| **查看历史** | 查看所有备份记录（名称、时间、表数、行数、大小） |
| **下载** | 下载备份文件到本地 |
| **恢复** | 将备份恢复到 Memory 层 |
| **删除** | 删除不需要的备份 |

### 11.3 恢复流程

1. 选择要恢复的备份
2. 点击「恢复」
3. 确认恢复操作（会覆盖当前 Memory 配置）
4. 恢复后，前往「配置同步」页面 Apply 到 Runtime

### 11.4 与向导备份的区别

- 向导 W48/W49：导出/导入 JSON 格式的配置文件
- 配置备份页面：内部快照管理，支持创建、下载、恢复、删除

## 12. 集群管理

### 12.1 功能概述

集群管理用于管理 ProxySQL 原生集群，支持多节点配置同步和状态监控。

### 12.2 主要功能

- **创建/编辑/删除集群组**
- **添加/移除集群节点**
- **自动发现节点**：读取 `proxysql_servers` 表
- **跨节点配置同步**：选择源节点和目标节点，同步指定模块
- **集群变量配置**：批量配置 `admin-cluster_*` 系列变量
- **集群状态监控**：在线状态、版本、运行时间、配置一致性
- **同步审计日志**

### 12.3 配置同步流程

```
源节点 RUNTIME → 目标节点 MEMORY → LOAD TO RUNTIME → SAVE TO DISK
```

## 13. 用户管理（管理员）

### 13.1 功能概述

管理员可以创建、编辑、启用/禁用和删除 WebUI 的登录用户。

### 13.2 用户角色

| 角色 | 权限 |
|------|------|
| **Admin（管理员）** | 完整权限：配置管理、用户管理、集群管理、备份恢复 |
| **Operator（操作员）** | 操作权限：配置管理、集群管理，不可管理用户 |
| **Viewer（只读）** | 只读权限：查看仪表盘、表浏览器、配置，不可修改 |

### 13.3 操作

- **添加用户**：填写用户名、密码、邮箱、角色
- **编辑用户**：修改角色、邮箱、密码
- **启用/禁用**：临时禁止用户登录
- **删除用户**：永久移除用户账号

## 16. 故障排除

### 16.1 常见连接问题

| 问题 | 可能原因 | 解决方案 |
|------|---------|---------|
| 无法连接 ProxySQL | 网络不通或端口错误 | 检查 ProxySQL 是否运行，确认管理端口（默认 6032） |
| 认证失败 | 用户名或密码错误 | 检查 ProxySQL 管理用户凭证。注意 admin 用户仅限本地连接 |
| 连接超时 | 网络延迟或防火墙 | 检查防火墙规则，确认端口开放 |
| WebSocket 断开 | 网络不稳定 | 等待自动重连，或刷新页面 |
| 数据库管理连接失败 | 后端凭据不匹配 | 确认 `mysql_users` 中用户密码与后端 MySQL 一致 |

### 16.2 ProxySQL 连接问题

1. 确认 ProxySQL 服务正在运行：
   ```bash
   systemctl status proxysql
   ps aux | grep proxysql
   ```

2. 测试管理端口连通性：
   ```bash
   mysql -h 127.0.0.1 -P 6032 -u admin -p
   ```

3. 检查 ProxySQL 日志：
   ```bash
   tail -f /var/lib/proxysql/proxysql.log
   ```

### 16.3 配置同步失败

| 症状 | 原因 | 解决方案 |
|------|------|---------|
| Apply 失败 | MEMORY 中有无效配置 | 检查表数据是否有冲突或无效值 |
| Save 失败 | 磁盘空间不足或权限问题 | 检查 `/var/lib/proxysql/` 权限和空间 |
| 配置不一致 | 手动修改了 ProxySQL | 使用配置差异对比功能排查 |

### 16.4 日志位置

- **应用日志**：Docker 容器标准输出（`docker compose logs`）
- **ProxySQL 日志**：`/var/lib/proxysql/proxysql.log`
