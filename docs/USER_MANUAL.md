## 1. 快速入门

### 1.1 什么是 ProxySQL Admin WebUI

ProxySQL Admin WebUI 是一个现代化的 **Web 图形化管理界面**，用于管理 [ProxySQL](https://proxysql.com/) 数据库代理服务器。它通过 ProxySQL 的 MySQL 管理端口（默认 6032）提供配置可视化、实时监控和运维操作。

**核心能力：**

- 无需手写 SQL 即可完成配置（通过 63 个向导）
- 实时监控连接数、QPS、连接池等关键指标
- 管理多个 ProxySQL 实例
- 支持 MySQL 和 PostgreSQL 后端
- 完整的 DISK/MEMORY/RUNTIME 三层配置同步

### 1.2 系统要求

| 项目 | 最低要求 | 推荐配置 |
|------|---------|---------|
| 操作系统 | Linux (任何发行版) | Ubuntu 22.04+ / Debian 12+ |
| Python | 3.10+ | 3.12+ |
| Node.js | 24+ (仅开发模式) | 24+ (latest LTS) |
| 内存 | 512 MB | 2 GB+ |
| 磁盘 | 100 MB | 1 GB+ |
| ProxySQL | 2.7+ | 3.0+ |

### 1.3 安装方式

#### 方式一：Docker 部署（推荐生产环境）

```bash
cd proxysql-admin-webui
make docker-build
make docker-up

# 访问 http://localhost:8080
```

Docker Compose 将同时启动 ProxySQL 实例和管理界面。

#### 方式二：裸机运行（推荐开发环境）

```bash
# 1. 安装 Python 依赖
make install

# 2. 构建前端（首次或前端改动后）
make build-frontend

# 3. 启动服务（单进程）
make run

# 访问 http://localhost:8080
```

#### 方式三：开发模式（热重载）

```bash
make install

# 终端 1：后端热重载 (:8080)
make dev-backend

# 终端 2：前端 Vite 热重载 (:5173)
make dev-frontend

# 访问 http://localhost:5173
```

### 1.4 首次登录与修改密码

1. 打开浏览器访问 `http://localhost:8080`
2. 使用默认凭证登录：
   - 用户名：`admin`
   - 密码：`admin`
3. **强烈建议**：首次登录后立即修改密码。
   - 点击右上角用户头像 →「修改密码」
   - 输入当前密码和新密码
   - 新密码需满足复杂度要求（至少 8 位，包含大小写字母和数字）

## 2. 仪表盘

### 2.1 理解仪表盘指标

仪表盘是登录后的默认页面，展示 ProxySQL 实例的实时监控数据：

| 指标 | 说明 | 数据来源 |
|------|------|---------|
| 客户端连接数 | 当前活跃的客户端连接 | `stats_mysql_global.Client_Connections_connected` |
| QPS（每秒查询数） | 当前查询吞吐量 | `stats_mysql_global.Questions` |
| 连接池使用率 | ConnUsed / (ConnUsed + ConnFree) | `stats_mysql_connection_pool` |
| 内存使用 | ProxySQL 进程内存 | `stats_memory_metrics` |
| 后端服务器状态 | 在线/离线服务器数量 | `mysql_servers` + `stats_mysql_connection_pool` |

### 2.2 服务器选择器

位于页面顶部的服务器选择器用于切换管理的 ProxySQL 实例：

- 下拉菜单显示所有已添加的服务器
- 切换服务器后，所有页面数据自动刷新
- 底部状态栏显示当前连接状态

### 2.3 实时监控

- 仪表盘数据通过 **WebSocket** 实时推送
- 数据刷新频率：默认每 5 秒
- 如果 WebSocket 断开，页面会显示重连提示

### 2.4 WebSocket 连接状态

底部状态栏右侧显示 WebSocket 连接状态图标：

- **绿色**：已连接，实时数据推送正常
- **黄色**：正在连接或重连中
- **红色**：连接断开

## 3. 服务器管理

### 3.1 添加 ProxySQL 服务器

1. 点击侧边栏「服务器管理」
2. 点击「添加服务器」按钮
3. 填写以下信息：
   - **名称**：给服务器起一个易于识别的名称（如"生产环境"）
   - **主机地址**：ProxySQL 管理接口的 IP 地址
   - **端口**：管理端口（默认 6032）
   - **用户名**：ProxySQL 管理员用户名
   - **密码**：ProxySQL 管理员密码
4. 点击「测试连接」验证连通性
5. 点击「保存」

### 3.2 编辑服务器连接信息

1. 在服务器列表中，点击目标服务器的「编辑」按钮
2. 修改需要更改的字段
3. 点击「保存」

### 3.3 测试连接

1. 在服务器列表中，点击目标服务器的「测试连接」按钮
2. 系统将尝试连接到 ProxySQL 管理端口
3. 显示连接结果：成功/失败及耗时

### 3.4 删除服务器

1. 在服务器列表中，点击目标服务器的「删除」按钮
2. 确认删除操作
3. 注意：删除服务器不会影响 ProxySQL 实例本身，仅从管理界面移除

### 3.5 多实例管理

- 可添加任意数量的 ProxySQL 服务器
- 通过服务器选择器快速切换
- 每个服务器的配置和监控数据独立管理

## 4. 配置向导

### 4.1 什么是向导

配置向导是一种**引导式表单工具**，帮助您通过填写表单来完成 ProxySQL 配置，无需手写 SQL 语句。系统内置 **63 个向导（W01-W63）**，覆盖 8 大类别。

### 4.2 向导类别概览

| 类别 | 向导编号 | 说明 |
|------|---------|------|
| 后端服务器管理 | W01-W08 | 添加/编辑/批量导入 MySQL/PgSQL 后端服务器、SSL 配置、连接测试 |
| 后端用户管理 | W09-W15 | 创建/编辑/管理后端用户、LDAP 映射、前后端分离 |
| 查询路由规则 | W16-W23 | 读写分离、缓存/重写/镜像规则、超时限流、日志规则 |
| 复制与集群拓扑 | W24-W28 | 主从复制、Group Replication、Galera、Aurora、PgSQL 复制 |
| 系统配置 | W29-W42 | 连接池/查询处理/缓存/监控参数、Admin 用户、集群节点/同步参数 |
| 防火墙与安全 | W43-W45 | 用户/规则白名单、SQL 注入防护 |
| 运维与配置同步 | W46-W52 | Apply All、Save All、配置备份/恢复、磁盘加载、重置统计 |
| 监控与诊断 | W53-W63 | 查询分析、命令/规则统计、连接池/进程/状态面板、集群状态 |

> 完整的向导参考请参阅 [配置向导完整指南](./WIZARD_GUIDE.md)。

### 4.3 使用向导的流程

每个向导遵循统一的操作流程：

1. **选择向导**：在向导列表中选择需要的向导
2. **填写表单**：根据向导提示填写配置参数
3. **预览 SQL**：系统根据表单内容自动生成对应的 SQL 语句
4. **执行**：确认无误后执行 SQL

### 4.4 自动 Apply 和自动 Save

向导提供了两个便捷选项：

- **自动 Apply（LOAD TO RUNTIME）**：执行向导后自动将变更应用到运行时
- **自动 Save（SAVE TO DISK）**：执行向导后自动将变更持久化到磁盘

启用这两个选项后，无需手动执行配置同步操作。建议在生产环境中谨慎使用自动 Apply，先预览 SQL 确认无误。

### 4.5 查看向导执行历史

- 在向导页面中，可以查看历史执行记录
- 每次执行记录包含：向导名称、执行时间、SQL 语句、执行结果
- 可用于审计和回滚参考

### 4.6 常见向导工作流

#### 工作流 1：添加后端 MySQL 服务器

1. 使用 **W01（添加 MySQL 后端服务器）** 添加单个服务器
2. 或使用 **W03（批量导入后端服务器）** 批量添加
3. 使用 **W08（后端连接测试）** 验证连接
4. 如需要 SSL，使用 **W06（后端 SSL 参数）** 配置

#### 工作流 2：设置读写分离

1. 使用 **W16（读写分离快速设置）** 一键创建读写分离规则
2. 使用 **W24（主从复制配置）** 注册读写主机组
3. 根据需要调整规则，使用 **W17（添加查询路由规则）**

#### 工作流 3：创建 MySQL 用户

1. 使用 **W09（创建 MySQL 后端用户）** 创建新用户
2. 使用 **W12（修改用户密码）** 设置密码
3. 使用 **W13（启用/禁用用户）** 管理用户状态

#### 工作流 4：配置查询规则

1. 使用 **W17（添加查询路由规则）** 创建基本路由规则
2. 使用 **W18（查询缓存规则）** 添加缓存
3. 使用 **W20（查询超时/限流规则）** 设置超时
4. 使用 **W23（查询日志规则）** 启用查询日志

#### 工作流 5：设置复制拓扑

1. 使用 **W24（主从复制）** 配置传统异步复制
2. 或使用 **W25（Group Replication）** 配置组复制
3. 或使用 **W26（Galera 集群）** 配置 Galera
4. 或使用 **W27（AWS Aurora）** 配置 Aurora 集群

## 5. 表浏览器

### 5.1 浏览 ProxySQL 配置表

表浏览器允许您直接查看和编辑所有 ProxySQL 配置表的数据：

1. 点击侧边栏「表浏览器」
2. 从下拉菜单中选择目标表（如 `mysql_servers`、`mysql_users`）
3. 表格将显示该表的所有数据行

### 5.2 搜索和过滤

- **全局搜索**：在搜索框中输入关键词，系统会筛选匹配的行
- **搜索范围**：默认搜索所有文本列
- **实时过滤**：搜索结果实时更新

### 5.3 分页

- 默认每页显示 50 条记录
- 使用底部分页控件切换页码
- 支持调整每页显示条数

### 5.4 内联编辑

表浏览器支持直接在表格中进行数据编辑：

- **插入**：点击「添加行」按钮，在新行中填写数据后保存
- **更新**：双击目标单元格进行编辑，按 Enter 确认
- **删除**：选中行，点击「删除」按钮

> 注意：编辑操作直接在 ProxySQL 的 MEMORY 层生效。要使变更持久化，请使用配置同步功能。

### 5.5 导出说明

表浏览器当前版本不提供直接导出功能。如需导出数据，请使用 SQL 查询控制台执行 `SELECT * FROM <table>` 并手动复制结果。

## 6. SQL 查询控制台

### 6.1 专家模式概述

SQL 查询控制台是一个面向高级用户的**专家模式**，允许您直接对 ProxySQL 管理端口执行任意 SQL 语句。

> **警告**：直接执行 SQL 可能绕过系统的安全检查。请确保您理解每条 SQL 语句的作用。

### 6.2 选择查询目标

在执行 SQL 前，请选择查询目标：

| 目标 | 说明 | 示例 |
|------|------|------|
| Admin | ProxySQL 管理接口（默认 6032） | `SELECT * FROM mysql_servers` |
| MySQL | 通过 ProxySQL 转发到后端 MySQL | `SELECT * FROM information_schema.tables` |
| PgSQL | 通过 ProxySQL 转发到后端 PostgreSQL | `SELECT * FROM pg_stat_activity` |

### 6.3 执行查询

1. 在编辑器中输入 SQL 语句
2. 选择查询目标
3. 点击「执行」按钮或按 `Ctrl+Enter`
4. 查询结果在下方显示

### 6.4 查询历史

- 所有执行的查询自动保存到历史记录
- 可查看历史查询及其执行结果
- 可重新执行历史查询

### 6.5 安全注意事项

- **不要**执行 `DELETE`、`DROP` 等破坏性操作，除非您完全理解其影响
- 对生产环境执行修改前，建议先在测试环境验证
- 所有查询操作会被记录到审计日志

## 7. 配置同步

### 7.1 理解 DISK/MEMORY/RUNTIME 三层

ProxySQL 使用三层配置模型：

```
┌──────────┐
│  RUNTIME │  ← 实际生效的配置（内存中的运行时状态）
├──────────┤
│  MEMORY  │  ← 当前编辑的配置（内存中的工作副本）
├──────────┤
│   DISK   │  ← 持久化的配置（磁盘上的 SQLite 数据库）
└──────────┘
```

- **RUNTIME**：ProxySQL 当前正在使用的配置。修改 MEMORY 不会立即影响 RUNTIME。
- **MEMORY**：您在 WebUI 中编辑的配置。所有向导和表编辑器的修改都作用于此层。
- **DISK**：持久化存储。ProxySQL 重启后从此处加载配置。

### 7.2 应用配置变更（Apply）

将 MEMORY 层的修改推送到 RUNTIME 层，使其立即生效：

1. 点击「配置同步」
2. 查看需要同步的模块列表
3. 选择要应用的模块（或全选）
4. 点击「Apply to Runtime」

对应 SQL：`LOAD <module> TO RUNTIME`

### 7.3 保存到磁盘（Save）

将 RUNTIME 层的配置持久化到 DISK：

1. 在配置同步页面
2. 选择要保存的模块
3. 点击「Save to Disk」

对应 SQL：`SAVE <module> TO DISK`

### 7.4 放弃变更（Discard）

从 RUNTIME 恢复 MEMORY，放弃未应用的修改：

1. 选择要放弃的模块
2. 点击「Discard Changes」

### 7.5 从磁盘加载（Load）

从 DISK 加载配置到 MEMORY：

1. 选择要加载的模块
2. 点击「Load from Disk」

对应 SQL：`LOAD <module> TO MEMORY`

### 7.6 配置差异可视化

配置同步页面提供 DISK/MEMORY/RUNTIME 三层的差异对比：

- **行级对比**：显示各表中哪些行在层间不一致
- **颜色标记**：绿色（新增）、红色（删除）、黄色（修改）
- **模块状态**：每个模块显示是否有未同步的变更

## 8. 集群管理

### 8.1 创建集群组

ProxySQL 支持原生集群，多个 ProxySQL 节点可以自动同步配置：

1. 点击「集群管理」
2. 点击「创建集群组」
3. 填写集群名称和描述
4. 点击「创建」

### 8.2 添加集群成员

1. 进入集群详情页
2. 在「节点」标签页中点击「添加节点」
3. 选择已添加的 ProxySQL 服务器
4. 点击「添加」

### 8.3 监控集群节点状态

在集群详情页的「节点」标签页中查看：

| 状态 | 说明 |
|------|------|
| 在线 | 节点正常运行，连接正常 |
| 离线 | 节点不可达 |
| 版本 | ProxySQL 版本号 |
| 运行时间 | 节点已运行时长 |
| Checksum 一致性 | 配置校验和是否与集群一致 |

### 8.4 跨节点配置同步

1. 选择源节点（配置来源）
2. 选择目标节点（配置去向）
3. 点击「同步配置」
4. 系统将：源节点 RUNTIME → 目标节点 MEMORY → LOAD TO RUNTIME → SAVE TO DISK

### 8.5 集群变量配置

在集群详情页的「变量」标签页中配置 `admin-cluster_*` 系列变量：

- `admin-cluster_username` / `admin-cluster_password`：集群认证凭证
- `admin-cluster_check_interval_ms`：节点健康检查间隔
- `admin-cluster_mysql_servers_diffs_before_sync`：同步前差异阈值

### 8.6 查看同步审计日志

在集群详情页的「日志」标签页中查看所有同步操作的记录：

- 同步时间
- 源节点和目标节点
- 同步的模块
- 成功/失败状态

## 9. 用户管理（管理员）

> 此功能仅对 **Admin** 角色开放。

### 9.1 创建用户

1. 点击「用户管理」
2. 点击「创建用户」
3. 填写用户信息：
   - **用户名**：登录名
   - **密码**：需满足复杂度要求
   - **角色**：Admin / Operator / Viewer
4. 点击「创建」

### 9.2 角色说明

| 角色 | 权限 | 适用场景 |
|------|------|---------|
| **Admin** | 所有权限（含用户管理） | 系统管理员 |
| **Operator** | 配置修改、执行向导、表编辑 | 运维人员 |
| **Viewer** | 仅查看（仪表盘、表浏览器、监控） | 只读监控人员 |

### 9.3 编辑用户信息

1. 在用户列表中点击目标用户的「编辑」
2. 修改角色或其他信息
3. 点击「保存」

### 9.4 禁用用户

1. 在用户列表中点击目标用户的「禁用」
2. 禁用后该用户将无法登录
3. 可随时重新启用

## 11. 故障排除

### 11.1 常见连接问题

| 问题 | 可能原因 | 解决方案 |
|------|---------|---------|
| 无法连接 ProxySQL | 网络不通或端口错误 | 检查 ProxySQL 是否运行，确认管理端口（默认 6032） |
| 认证失败 | 用户名或密码错误 | 检查 ProxySQL 管理用户凭证 |
| 连接超时 | 网络延迟或防火墙 | 检查防火墙规则，确认端口开放 |
| WebSocket 断开 | 网络不稳定 | 等待自动重连，或刷新页面 |

### 11.2 ProxySQL 连接问题

1. 确认 ProxySQL 服务正在运行：
   ```bash
   systemctl status proxysql
   # 或
   ps aux | grep proxysql
   ```

2. 测试管理端口连通性：
   ```bash
   mysql -h 127.0.0.1 -P 6032 -u admin -p
   ```

3. 检查 ProxySQL 日志：
   ```bash
   tail -f /var/lib/proxysql/proxysql.log
   ```

### 11.3 配置同步失败

| 症状 | 原因 | 解决方案 |
|------|------|---------|
| Apply 失败 | MEMORY 中有无效配置 | 检查表数据是否有冲突或无效值 |
| Save 失败 | 磁盘空间不足或权限问题 | 检查 `/var/lib/proxysql/` 权限和空间 |
| 配置不一致 | 手动修改了 ProxySQL | 使用配置差异对比功能排查 |

### 11.4 日志位置

- **应用日志**：`backend/logs/` 目录
- **ProxySQL 日志**：`/var/lib/proxysql/proxysql.log`
- **审计日志**：WebUI 的「系统设置」→「审计日志」页面
